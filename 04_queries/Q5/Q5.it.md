# Q5 – Miglioramento degli stabilimenti nel tempo

## Domanda di analisi

Gli stabilimenti sottoposti a ispezione sanitaria **mostrano un miglioramento nel lungo periodo**
oppure tendono a mantenere (o peggiorare) il proprio livello di qualità?

L’obiettivo è misurare l’**efficacia complessiva delle azioni correttive**
e del sistema di controllo sanitario, confrontando il punteggio iniziale e finale di ciascun stabilimento.

## Contesto analitico

L’analisi utilizza il **modello dati a stella**, in particolare:

- **Fact table:** `inspection_events_table`
- **Dimensioni:**
  - `establishment_dim` (identificazione univoca degli stabilimenti)
  - `date_dim` (ordinamento temporale delle ispezioni)

La metrica osservata è:
- `score_assigned` (punteggio assegnato all’ispezione)

> Nota: un **punteggio più basso indica condizioni migliori**, un **punteggio più alto indica maggiori criticità**.

## Logica dell’analisi

Per ciascun stabilimento:

1. si individua la **prima ispezione** disponibile nel dataset
2. si individua l’**ultima ispezione** disponibile
3. si confrontano i due punteggi:
   - `last_score < first_score` → **miglioramento**
   - `last_score ≥ first_score` → **nessun miglioramento**

L’analisi fornisce quindi una visione **longitudinale** del comportamento degli stabilimenti nel tempo, indipendentemente dal numero di ispezioni subite.

## Query SQL

La query utilizza:

- **CTE**
- **funzioni finestra** (`FIRST_VALUE`)
- aggregazioni finali

```sql
WITH scores_list AS (
    SELECT DISTINCT
        establishment_key,
        FIRST_VALUE(score_assigned) OVER (
            PARTITION BY establishment_key
            ORDER BY date_key ASC
        ) AS first_score,
        FIRST_VALUE(score_assigned) OVER (
            PARTITION BY establishment_key
            ORDER BY date_key DESC
        ) AS last_score
    FROM
        inspection_events_table
),
final_counts AS (
    SELECT
        COUNT(DISTINCT establishment_key) AS total_establishments,
        COUNT(
            CASE
                WHEN last_score < first_score THEN 1
            END
        ) AS improved_establishments
    FROM
        scores_list
)
SELECT
    ROUND(
        (CAST(improved_establishments AS DECIMAL) / total_establishments) * 100,
        2
    ) AS improved_percentage
FROM
    final_counts;
```

## Output

Il risultato finale è stato esportato in formato CSV:

* `improved_%.csv`

## Risultati principali

| Categoria di stabilimenti | Criterio SQL | Percentuale |
|:-:|:-:|:-:|
| 🟢 Stabilimenti migliorati | `last_score < first_score` | **22%** |
| 🔴 Stabilimenti non migliorati | `last_score ≥ first_score` | **78%** |

- Circa il 22% degli stabilimenti mostra un **miglioramento netto** tra la prima e l’ultima ispezione
- Il restante **78%**:
  - mantiene un livello simile
  - oppure mostra un peggioramento

## Insight

L’analisi evidenzia che:

* il sistema di ispezioni **produce miglioramenti**, ma solo per una minoranza
* molti stabilimenti:

  * correggono temporaneamente le criticità
  * ma non mostrano un miglioramento strutturale nel lungo periodo

Questo suggerisce che:

* le azioni correttive sono spesso efficaci **nel breve termine**
* ma non sempre inducono un cambiamento duraturo
  nei processi interni degli esercizi
